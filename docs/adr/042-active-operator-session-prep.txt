상태: Accepted
날짜: 2026-07-19
================================================================================

Context
--------------------------------------------------------------------------------
set_current_operator_id() is transaction-local. OperatorContext RPC from the
browser does not stick for later PostgREST calls. Staff SELECT still sees all
tenants via is_staff() / rls_is_staff_session_ok(). Before locking staff RLS to
current_operator_id(), the active tenant must reach every API request reliably.


Decision
--------------------------------------------------------------------------------
1. Cookie tms_active_operator_id (mirrors localStorage on switch/refresh)
2. Middleware stamps x-active-operator-id from cookie
3. requireStaffApiAuth + resolveFinancialApiAuth call
   applyActiveOperatorSession() → set_current_operator_id RPC on that client
4. pilot-status returns activeOperatorSession cookie prep status
5. NO staff RLS rewrite in this phase


Non-goals
--------------------------------------------------------------------------------
- JWT app_metadata.operator_id write
- operator_id = current_operator_id() staff lock-down
- Booking/payment logic changes

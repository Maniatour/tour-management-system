Context
--------------------------------------------------------------------------------
company_expenses (6b.3) is tenant-scoped. Tour and reservation expense tables
still listed the shared ledger, so Operator B with Ops ON could see Kovegas
rows via admin expense managers and ExpenseReportTab.


Decision
--------------------------------------------------------------------------------
1. Migration: 20260719180000_saas_operator_id_tour_reservation_expenses.sql
   - tour_expenses.operator_id NOT NULL (from tours / Kovegas)
   - reservation_expenses.operator_id NOT NULL (from reservations, then tours,
     then Kovegas)
   - Views *_no_statement_match pick up the column via SELECT *

2. App list/create stamp (MUST):
   - TourExpenseManager / TourReceiptModal / AllTourExpensesManager
   - GET/POST /api/reservation-expenses (+ suggestions) + ReservationExpenseManager
   - ExpenseReportTab filters tour + reservation by operator_id
   - Sync services stamp operator_id on insert/upsert
   - Statement bulk/adjustment insert paths stamp operator_id

3. Deferred:
   - Full StatementReconciliationTab / CashLedger rewrite
   - CategoryManagerModal normalize, expense-unified-duplicate-scan
   - Parent-id-only readers (PricingSection, etc.)


Non-goals
--------------------------------------------------------------------------------
- Booking / payment / checkout changes
- ops_* schema move
- Changing expense business logic


Next
--------------------------------------------------------------------------------

Context
--------------------------------------------------------------------------------
Fleet (6b.0–6b.1) and attendance (6b.2) are tenant-scoped. Company expenses
(internal Ops finance) still listed the shared table, so a second Ops-enabled
tenant could see Kovegas ledger rows via /admin/expenses.


Decision
--------------------------------------------------------------------------------
1. Migration: 20260719150000_saas_operator_id_company_expenses.sql
   - ADD company_expenses.operator_id NOT NULL (default Kovegas)
   - Backfill all rows → Kovegas
   - FK + index
   - View company_expenses_no_statement_match (SELECT ce.*) picks up column

2. App list/create stamp:
   - GET/POST /api/company-expenses?operatorId=
   - CompanyExpenseManager passes active operatorId
   - Auto-create from vehicle maintenance stamps vehicle/maintenance operator_id
   - Expense sync from company_expenses filters by operator_id

3. Non-goals this slice:
   - reservation_expenses / tour_expenses operator_id
   - Full reconciliation / cash-ledger rewrite (still Kovegas-first; rows
     remain Kovegas after backfill so behavior unchanged for Tenant #1)
   - ops_* schema move
   - Booking / payment / checkout changes
